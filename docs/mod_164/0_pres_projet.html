1. Documentation du module 164 (29.03.2024)

1.1. But

Le but de ce module 164 est de faire un CRUD (Create Read Update Delete) sur les tables d’une BD.

  • Ma démo consiste en une interface (HTML, FLASK, Python, etc.) et une BD MySql.

    • Il s’agit d’un gestionnaire de films et leurs genres associés.

    • Soit un système CRUD (Create Read Update Delete) presque complet sur les 3 tables de la BD films : Soit “t_film”; “t_genre” et la table intermédiaire “t_genre_film”

1.2. Votre travail

  • Installer tous les programmes nécessaires (Logiciels pour faire fonctionner la démo)

  • Faire fonctionner la démonstration « Films » (Mode d’emploi de la démonstration « Films »)

  • Faire une copie de la démonstration (Copier mon projet « Films » (29.03.2024))

  • Importer votre base de données (DUMP) dans votre copie (Importation de votre DUMP MySql)

  • Connecter votre BD avec vos paramètres du fichier « .env » (Modifier le fichier « .env » des paramètres)

  • Dans le fichier « gestion_genres_crud.py »(Ligne 37,46,50) sur « t_genre », il faut utiliser vos requêtes de votre table (Fichier Python)

  • Modifier la page html « genres_afficher.html »(Lignes 37,40,43) pour qu’elle affiche les champs de votre BD (Fichier HTML)

  • Lire le chapitre Évaluation de votre module 164
